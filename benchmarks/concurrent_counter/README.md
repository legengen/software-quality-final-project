# Concurrent Counter Benchmark

This tiny project is a custom concurrency repair benchmark for the AGENTLESS reproduction.

The base implementation intentionally contains a race condition in `ConcurrentCounter.increment`.
Sequential behavior works, but concurrent increments can lose updates because the method performs
a read-modify-write sequence without synchronization.

The AGENTLESS case resources are generated by:

```bash
python scripts/generate_custom_concurrency_case.py
```

The generated `test_patch` adds the failing concurrency test. The local evaluator applies that
test patch and a model-generated patch, then runs `pytest`.
